# ZEREM (זרם) — מפת דרכים ורעיונות

מסמך עבודה חי. מה שסומן ✅ כבר באוויר; השאר לפי סדר עדיפות משוער.

## גל 1 — חשבוניות (מתחיל ביום ראשון)
- [ ] קבלת API של SmartBee ופיילוט על החשבון של סתיו.
- [ ] **מגבלה: חשבונית עד ₪5,000** — מעל זה נדרשת אסמכתא מרשות המיסים; המערכת תחסום/תתריע.
- [ ] ארכיטקטורה: שכבת ספקים גנרית (כמו שכבת ה-AI) — SmartBee ראשון, אחר כך חשבונית ירוקה / iCount וכו'.
- [ ] כל משתמש מחבר את חשבון החשבוניות *שלו* (מפתח אישי בהגדרות).
- [ ] **אזור נפרד לגמרי במערכת**: הניווט יתחלק ל-3 עולמות — הגדרות | פרויקטים | ניהול חשבונות. הרבה עסקים קטנים עובדים עם רו"ח — עולם החשבונות חייב להיות מופרד ואופציונלי.
- [ ] הזרימה: הצעה בסטטוס "הושלם" ← כפתור "הפק חשבונית" ← סטטוס "שולם".

## גל 2 — ספקים והשוואת מחירים (החזון הגדול)
- [ ] **מודול "ניהול ספקים" באדמין בלבד (מפרט מסתיו, 04/07)**: לכל ספק — כרטיס פרטים (שם, כתובת, טלפון, מייל, קישור לאתר) + העלאת כל המוצרים והמחירים שלו (ייבוא CSV/הדבקה כמו במאגר). המאגרים נשמרים פר-ספק (KV `supplier:<id>`), ובהמשך שלב התמחור משווה ביניהם. לתכנן מבנה לפני קוד.
- [ ] שותפויות ספקים: ארכה, חשמל ישיר ועוד — קבלת מחירונים רשמיים; מתחילים עם הספק השכונתי של סתיו (רשימת מוצרים ב-PDF ← ייבוא).
- [ ] שלב "תמחר" משווה בין ספקים ומציג מאיפה הכי משתלם לקנות.
- [ ] כפתור "הוסף את כל המוצרים מאתר X" (הרחבת הסריקה לזחילה מלאה — דרך שותפות/פיד, לא scraping).
- [ ] "מצא את הספק הקרוב אליך" + כתובת/עיר מהמשתמש (דרך Google) + זמן נסיעה (Waze/Maps API) — עתיד רחוק.
- ✅ CTA בדף הנחיתה: "אתה ספק? הצטרף" (וואטסאפ/מייל) + טופס "מכיר ספק? שם + עיר" (מגיע למייל של סתיו).
- [ ] לוגואים של ספקים שותפים בדף התדמית של זרם.

## גל 3 — ניהול המאגר (אדמין)
- ✅ כפתור "נתח וסדר את המאגר עם AI" בפאנל האדמין — מאחד וריאציות זניחות, שומר וריאציות הנדסיות, מסיר זבל.
- ✅ סביבת ניהול מאגרים באדמין: 1️⃣ העלאה (הדבקה/CSV) ← 2️⃣ "נתח את המאגר" ← 3️⃣ "המאגר של כולם" (רשימה + פרסום).
- ✅ זיהוי אוטומטי של שינוי במאגר ← התראה "זוהה שינוי — נתח ופרסם".
- [ ] קטגוריות: הדף יתחלף בין חשמל / אינסטלציה / מיזוג וכו' (הכנה להתרחבות לענפים).
- [ ] **רשימת משתמשים באדמין (סתיו, 04/07 — "צריך לזכור")**: "ניהול מסלולים" מציג היום רק חיפוש לפי אימייל; להוסיף רשימה של כל המשתמשים הרשומים (KV list על `user:*`) עם המסלול של כל אחד. הערה: משתמש מופיע ב-KV רק אחרי שסינכרן נתונים לפחות פעם אחת (sj.eng.business@gmail.com יופיע אחרי כניסה+שמירה ראשונה).

### מדיניות איחוד וריאציות (הוחלט)
- מאחדים כשההבדל זניח מחירית (~עד 7%) ולא משנה בחירה הנדסית: "וו תליה פתוח/סגור" ← "וו תליה" במחיר הגבוה.
- לא מאחדים כשהווריאציה היא החלטה הנדסית: לוח 12/18/24 מודול, חתך כבל, אמפראז'.
- ביניים (עוגן ב-4 אורכים, פער 3.2–4.2): מאחדים לשם גנרי עם טווח או מחיר ממוצע-גבוה — ל-AI זה מספיק לתמחור; הבחירה המדויקת נעשית בשטח.
- שינוי בחירה של המשתמש (לוח 24 ← 18) נעשה **דרך הצ'אט** ("תחליף ל-18 מודול") — בלי UI ייעודי בשלב זה.
